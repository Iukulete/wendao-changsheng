# Agent Bridge 事件日志复盘

分析对象：GitHub Actions run `28999330696` 上传的 `wendao-agent-playthrough-report`，主要查看 `agent_playthrough_report.txt` 与 `agent_trace.log`。

## 这轮日志说明

- Agent Bridge 已经能真实进入菜单、开局、修炼、历练、事件选择、突破、进入道祖阶段。
- 事件总触发约 265 次，去重标题约 41 个。
- 高频事件集中在少数几类：`【传承】立下道标` 约 62 次、`【纪元】古修遗响` 约 30 次、`【因果】照雪战帖` 约 23 次、`【危机】人情逼试` 约 20 次。
- v0.2 注入的若干开局事件已经进入事件池，但在 Windows PowerShell 5.1 执行 `.ps1` 时出现 UTF-8 无 BOM 脚本被系统 ANSI 码页读取的问题，日志中可见部分标题变成乱码，例如 `【人情】清冷师姐旁观`。
- 道祖阶段卡住的原因不是事件池问题，而是 Agent 策略误把 `heavenlyDaoReady=true` 理解成“可以立刻突破天道”；实际仍需要道祖九层并攒满修为。

## 可改进点

1. **先修编码链路**：构建脚本优先用 `pwsh` 执行补丁，避免 Windows PowerShell 5.1 把中文 here-string 读成乱码。
2. **扩事件池**：现有 41 个标题支撑 265 次历练仍会明显重复，尤其传承、纪元、因果类高频。v0.3 先新增 24 条原创本地事件，覆盖开局、外院、家世、旧玉、空阙、坊市、同辈、师承、宗门、道途、灵宝、仙闻、鸿蒙碎梦等方向。
3. **降低单事件刷屏感**：新增事件不只给奖励，也提供线索、人情、旧债、空阙、道心与灵宝残印的小推进，减少“同一个标题反复出现”的感觉。
4. **保留版权边界**：新增事件只使用通用修仙叙事元素与本项目自有钩子，不引用外部小说人物、地名、势力名、专有体系或桥段。
5. **Agent 策略单独修**：道祖门槛解锁后仍继续修炼/闭关/少量历练，直到道祖九层且修为足够，再按突破。
