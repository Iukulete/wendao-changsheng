# 10,000 局自动试玩报告

日期：2026-07-23

引擎：Godot 4.7.1 stable official

发布候选：章节化叙事、独立结果页、命途长卷、目标化玩法与能力牌视觉层

## 范围

本轮只验证可执行玩法、状态持久化和异常边界，不执行产品美术审计。美术身份图与
剧情分镜继续按 `docs/ART_PRODUCTION_BRIEF_V1.md` 由独立任务制作。

10,000 局使用连续全局索引 `0..9999`，对应种子：

```text
73000001 + index * 7919
```

六种轮换模式共同覆盖：

- 六纪元新生、世界初始化、创作事件与连续剧情选择；
- 章节抬头、前情摘要、事件结果页、96 章有界长卷及其存档往返；
- 三式运功、阶段命途、确定性突破、限时敌踪、普通战斗与镜湖秘境完整结算；
- 存档写入、校验读取与关键游标往返；
- 当前世封存、下一世开始和跨世世界推进；
- 本地 AI 不可用时的确定性三选项降级；
- 每次关键动作后的生命值、灵力、资源、时代、世界年份和互斥战斗状态不变量。

其中每 10 局执行一次真实存档往返，共 1,000 次；每 17 局检查一次本地 AI
降级，共 589 次；轮回模式共 3,333 局。两个分片使用隔离的临时目录和不重叠种子。

## 结果

```text
shard offset 0:    5000/5000 passed, 0 failures
shard offset 5000: 5000/5000 passed, 0 failures
PLAYTEST_BATCH_OK: 10000 sessions across 2 shards at offset 0
```

两个分片耗时分别为 310.193 和 308.194 秒，单分片吞吐为 16.119 和 16.224 局/秒。
没有发现可复现的玩法或状态损坏问题；两个分片的错误计数、失败样本和运行时错误均为零。
本次运行使用最终能力牌视觉层与发布前音频清单，不复用旧报告的结果。

本次目标化玩法遥测为：普通战斗 4,852 胜、147 负，胜率 97.06%，平均 5.787
回合，平均损失 30.5742% 最大气血；288 次进入濒死阈值，4,976 次触发敌人第二阶段，
4,679 次完成反制节拍，3,541 次使用反制爆发。秘境完成 4,746 次、失败 223 次，
完成率 95.51%；轮回 3,333 次，真实存档往返 1,000 次，本地 AI 降级 589 次，
章节触发 10,000 次。战斗动作分布为攻击 8,448、守势 7,430、术法 12,591、
丹药 461，说明试玩覆盖了多种操作而不是单一按钮路径。

## 补充回归与修复

10,000 局状态模拟完成后，非美术 Godot 回归发现 UI 高频音效在首次冷加载超过
55 毫秒时可能绕过防抖：旧实现从“开始加载音频”的时刻计算冷却，第二次点击发生
时冷却可能已经耗尽。`AudioDirector.play_event()` 现在只在音频成功解析并接受播放
后写入冷却时间。

修复后音频系统用例连续运行 5 次全部通过，非美术 Godot 全量回归全部通过；
52 个自动 OpenGL 页面状态在 `800x720`、`1280x720`、`1440x900` 和
`1920x1080` 下均非空且互不重复，新增覆盖事件结果页与命途长卷的桌面、窄屏状态。
章节日志还单独验证了字段完整性、96 章上限、前情生成、旧档补齐和真实存档往返。
该布局检查不评价或筛选美术内容。

## 复跑

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\run_playtest_10k.ps1 -Count 10000 -Shards 2
```

使用 `-Offset 10000` 可运行下一段不重叠种子；失败输出会包含局号、模式、种子和
结构化详情。
