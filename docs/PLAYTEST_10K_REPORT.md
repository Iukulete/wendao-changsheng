# 10,000 局自动试玩报告

日期：2026-07-21

引擎：Godot 4.7.1 stable official

发布候选：阶段命途、上下文敌踪与三式运功改造

## 范围

本轮只验证可执行玩法、状态持久化和异常边界，不执行产品美术审计。美术身份图与
剧情分镜继续按 `docs/ART_PRODUCTION_BRIEF_V1.md` 由独立任务制作。

10,000 局使用连续全局索引 `0..9999`，对应种子：

```text
73000001 + index * 7919
```

六种轮换模式共同覆盖：

- 六纪元新生、世界初始化、创作事件与连续剧情选择；
- 三式运功、阶段命途、确定性突破、限时敌踪、普通战斗与镜湖秘境完整结算；
- 存档写入、校验读取与关键游标往返；
- 当前世封存、下一世开始和跨世世界推进；
- 本地 AI 不可用时的确定性三选项降级；
- 每次关键动作后的生命值、灵力、资源、时代、世界年份和互斥战斗状态不变量。

其中每 10 局执行一次真实存档往返，共 1,000 次；每 17 局检查一次本地 AI
降级，共 589 次；轮回模式共 3,333 局。四个分片使用隔离的临时目录和不重叠种子。

## 结果

```text
shard offset 0:    2500/2500 passed, 0 failures
shard offset 2500: 2500/2500 passed, 0 failures
shard offset 5000: 2500/2500 passed, 0 failures
shard offset 7500: 2500/2500 passed, 0 failures
PLAYTEST_BATCH_OK: 10000 sessions across 4 shards at offset 0
```

四个分片耗时分别为 103.759 到 104.781 秒，单分片吞吐为 23.859 到 24.094 局/秒。
没有发现可复现的玩法或状态损坏问题。测试工具开发期间发现 Windows PowerShell
在带超时的 `WaitForExit` 后可能尚未刷新子进程退出码；汇总器已改为再次等待并
刷新进程状态，避免把成功分片误报为失败。

本次目标化玩法遥测为：10,000 次敌踪出现，4,999 次被玩家迎战，4,471 次自然
过期；普通战斗 4,870 胜、129 负，胜率 97.42%，平均 5.29 回合，平均损失
23.32% 最大气血。阶段命途共完成 4,768 次，仅 5 次实战命途逾期；这 5 次均来自
有限期限内未选择迎战的有效失败路径，不是状态机异常。

## 补充回归与修复

10,000 局状态模拟完成后，非美术 Godot 回归发现 UI 高频音效在首次冷加载超过
55 毫秒时可能绕过防抖：旧实现从“开始加载音频”的时刻计算冷却，第二次点击发生
时冷却可能已经耗尽。`AudioDirector.play_event()` 现在只在音频成功解析并接受播放
后写入冷却时间。

修复后音频系统用例连续运行 5 次全部通过，18 组非美术 Godot 回归全部通过；
48 个自动 OpenGL 页面状态在 `800x720`、`1280x720`、`1440x900` 和
`1920x1080` 下均非空且互不重复。该布局检查不评价或筛选美术内容。
