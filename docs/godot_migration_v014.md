# Godot-only 迁移完成记录

## 结论

《问道长生》已切换到 Godot 4.7.1 stable 标准版。Godot 现在是唯一玩家运行时，旧 Win32/C++ 源码、入口、构建脚本和工作流已经从当前主线退役；需要追溯时可从删除提交之前的 Git 历史读取。

## 对等与替代

| 领域 | Godot 当前实现 |
|---|---|
| 成长 | 21 境界、九层、突破、失败代价、寿元与多类死亡闭环 |
| 轮回 | 前世、旧玉、传承、未竟因果、世界续时与十世长局 |
| 世界 | 数据化势力/NPC、关系、年度模拟、纪元切换和有界历史 |
| 物品 | 稳定 ID、背包、消耗、装备、锻造和轮回保留规则 |
| 战斗 | 确定性敌人意图、招式、状态、结算和中途恢复 |
| 叙事 | 外部事件数据、四条主线、四条跨世续章和校验器 |
| 成就 | 16 项成就、16 件永久玉兵、觉醒、蓄能与显圣 |
| 本地 AI | 本机进程桥、隐私摘要、成功/超时/禁用/非法输出回退 |
| 副本 | 可选镜湖秘境，能力牌组来自角色成长，不替代主循环 |
| 发布 | 固定 Godot、官方模板、Windows 导出、干净产物与 GitHub CI |

## 旧档边界

`LegacySaveImporter` 读取旧 Win32 `SAVE_V4/SAVE_V5` 文本格式，并将角色、世界、轮回、剧情与玉兵关键状态转换为统一 Save v2。导入器只以读取模式打开源文件；解析或校验失败时不会调用新版写盘逻辑，成功后也保留原文件。

## 完成证据

- 确定性十世长局覆盖自然寿尽、战斗死亡、飞升、道祖与天道。
- 本地 AI 成功、超时、禁用和非法输出四路径回归通过。
- `1280x720`、`1440x900`、`1920x1080` 与秘境两种界面通过真实 GL 截图验收。
- Windows release 导出和导出 EXE 后台冒烟通过。
- 发布包只有 `wendao-changsheng.exe` 与 `wendao-changsheng.pck`，测试和旧源码不会进入资源包。

后续不再以旧版菜单或实现为设计上限，内容、美术和交互演进以 [Godot-only 产品方向](godot_only_product_direction_v015.md) 为准。
