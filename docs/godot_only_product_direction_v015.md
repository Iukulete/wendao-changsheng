# v0.15 Godot-only 产品方向与删除门槛

## 定位

旧版功能对等是最低验收线，不是新版的设计上限。新版要把《问道长生》做成一款“世界会继续活、人物会记住选择、每一世都会改变下一世”的叙事修仙 Roguelike，而不是把旧 Win32 菜单原样翻译成 Godot 控件。

## 五个产品支柱

1. **世界不是背景**：玩家修炼时 NPC、宗门、资源、战争和时代同样推进；轮回后回到的是被上一世改变过的世界。
2. **长生不是数值条**：寿元制造真正的路线压力。突破、延寿、传承、守护他人和追逐权柄会争夺有限时间。
3. **道途由行为形成**：护生、凌霄、逆命、照因、造化、众生六个维度由长期选择塑造，改变事件、能力、美术反馈和轮回遗产。
4. **记忆比掉落更重要**：普通物品会失散，但人物关系、未竟因果、功法手感、器痕和世界传闻会跨世回响。
5. **时代拥有完整语言**：每个纪元不仅换背景，还要改变资源规则、NPC 目标、事件结构、声音、粒子、材质、排版和交互节奏。

## 功能对等矩阵

| 领域 | 旧版能力 | Godot 删除旧版前的最低条件 |
|---|---|---|
| 成长 | 21 境界、九层修炼、突破、寿元 | 稳定境界 ID、完整成长、失败代价、寿尽闭环与自动化测试 |
| 轮回 | 前世记录、传承、旧玉、未竟因果 | Save v2、世界续时、可解释遗响、跨世剧情钩子与多世长局 |
| 世界 | 15 名动态 NPC、目标、关系、死亡、世界事件 | 数据驱动 NPC/势力、年度模拟、关系记忆、时代资源与事件联动 |
| 物品 | 背包、丹药、材料、装备、锻造、玉兵、鸿蒙至宝 | 稳定物品 ID、装备/锻造状态机、轮回保留规则、UI 与回归测试 |
| 战斗 | 敌人、属性、胜负、奖励与战绩 | 可复现判定、招式/状态、敌人意图、事件接入和无阻断自动战斗 |
| 叙事 | 静态事件、连续剧情、第二世续章 | 外部数据、条件/效果类型、剧情图、冷却、跨世章节和校验器 |
| 本地 AI | llama.cpp 上下文、超时、模板回退 | Godot HTTP/进程桥、结构校验、成功/超时/禁用三路径与隐私边界 |
| 发布 | Win32 构建、多个旧 CI | Godot 唯一入口、Windows 导出、长局/截图/低配验证与干净产物 |

## 删除旧版的硬门槛

- 旧版核心功能全部有新版实现或明确被更好的设计替代，并有对应自动化验收。
- 至少完成 10 世确定性长局，覆盖自然寿尽、战斗死亡、飞升和道祖路线。
- 旧存档关键玩家、世界、轮回字段能够只读导入；失败时不覆盖原文件。
- 本地 AI 成功、超时、禁用和非法输出均能无损回退。
- 1280×720、1440×900、1920×1080 和无独显 GL Compatibility 环境通过。
- 导出包不含 C++ 源码、旧 EXE、旧补丁器或旧运行入口。

满足以上条件后，才删除 `src/`、`legacy_system/`、`world_system/`、`procedural_gen/`、旧 `build.bat`、旧启动入口和旧 Windows CI。删除提交必须可由前一版本追溯，但主线与发布包只保留 Godot 新版。
