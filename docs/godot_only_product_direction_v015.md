# Godot-only 产品方向

## 定位

《问道长生》不是旧菜单的换皮复刻，而是一款“世界会继续活、人物会记住选择、每一世都会改变下一世”的叙事修仙 Roguelike。

## 五个产品支柱

1. **世界不是背景**：玩家修炼时，NPC、宗门、资源、战争和时代同时推进；轮回后回到的是被上一世改变过的世界。
2. **长生不是数值条**：寿元制造路线压力，突破、延寿、传承、关系与权柄争夺同一段有限时间。
3. **道途由行为形成**：护生、凌霄、逆命、照因、造化、众生六个维度来自长期选择，并改变能力、事件和轮回遗产。
4. **记忆比掉落更重要**：普通物品可能失散，人物关系、未竟因果、功法手感、器痕和世界传闻会跨世回响。
5. **时代拥有完整语言**：纪元不仅更换背景，还要改变资源规则、人物目标、事件结构、声音、粒子、材质和交互节奏。

## 玩法边界

- 修炼、人物、世界和跨世叙事始终是主线。
- 普通战斗服务主线事件和成长，不被卡牌系统替代。
- 构筑玩法只在镜湖秘境、纪元遗迹和心魔劫等副本出现。
- 牌是角色功法、道途、玉兵、装备、关系、记忆和创伤的能力投影，不建立永久卡包经济。
- 本地 AI 只写受约束的叙事措辞，不裁定数值、奖励、死亡、存档或世界规则。

## 下一阶段

- 在每纪元六个自由历练事件的基础上继续扩展专属资源循环、势力目标和首领机制。
- 让关系、器痕与跨世定局产生更多可见的能力变化，而不是只增加数值。
- 增加角色身份稳定的立绘、场景变体、命中特效、音效和纪元材质语言。
- 优化秘境牌面信息层级、敌方演出、路线事件密度和能力来源说明。
- 持续用确定性长局、旧档迁移、真实 GL 截图和干净导出包约束迭代。
