# v0.17 产品级中文字体方向

## 目标

《问道长生》的字体需要同时提供修仙题材的文化气质与长时间游玩的阅读舒适度。艺术感只出现在足够大的展示标题中，正文、按钮、数值和状态信息必须以快速辨认、稳定对齐和低疲劳为第一优先级。

## 字体职责

- **Noto Serif SC** 只用于字号不小于 22 px 的展示标题，例如页面主标题、纪元名、首领名和重要章节题名。
- **Noto Sans SC** 用于正文、按钮、页签、提示、能力说明、对话、资源数值、战斗数值、状态标签和所有小于 22 px 的文本。
- 长句、连续段落、可点击文本、倒计时和变化频繁的数值不得使用 Noto Serif SC。
- 同一控件不混用两套字体。标题中的副标题、计数和状态仍使用 Noto Sans SC。
- 字体回退顺序必须显式配置；缺字时优先回退到覆盖对应字符的 Noto CJK 字体，不允许由操作系统随机替换为宋体、楷体或手写体。

## 字号 token

字号以最终屏幕上的物理像素为验收口径，不以 Godot 控件中未经缩放的名义数值代替。设计基准为 1440 x 900；1280 x 720 是必须完整支持的清晰度下限。

| Token | 字体 | 基准字号 | 720p 下限 | 用途 |
| --- | --- | ---: | ---: | --- |
| `type.display` | Noto Serif SC | 36 px | 30 px | 纪元揭示、首领登场等短时展示标题 |
| `type.page-title` | Noto Serif SC | 28 px | 24 px | 页面主标题、章节题名 |
| `type.section-title` | Noto Serif SC | 22 px | 22 px | 信息区块标题；宋体使用的绝对下限 |
| `type.heading` | Noto Sans SC | 20 px | 18 px | 面板标题、列表组标题、关键提示 |
| `type.value-large` | Noto Sans SC | 24 px | 22 px | 气血、寿元、伤害等主数值 |
| `type.value` | Noto Sans SC | 18 px | 18 px | 常规资源、回合、层数和状态数值 |
| `type.body-large` | Noto Sans SC | 18 px | 17 px | 重要叙事、确认信息、能力摘要 |
| `type.body` | Noto Sans SC | 16 px | 16 px | 正文、能力说明、对话和列表内容 |
| `type.button` | Noto Sans SC | 16 px | 16 px | 按钮、页签和所有可操作标签 |
| `type.meta` | Noto Sans SC | 14 px | 14 px | 来源、时间、次要状态和辅助信息 |
| `type.caption` | Noto Sans SC | 13 px | 13 px | 非关键图注；不得承载操作或规则 |

- 产品内不出现小于 13 px 的中文文本。
- 720p 下不得等比压缩整套界面字体；优先减少留白、调整布局、折行或滚动。
- 关键按钮、战斗信息和数值即使空间紧张，也不得低于各自 token 的 720p 下限。
- 用户界面缩放需要对最终像素取整，避免半像素采样导致笔画发虚。

## 字重与层级

- Noto Serif SC 展示标题使用 `600`；需要压场的首领名或纪元名可使用 `700`，正文区域不得用更重字重制造层级。
- Noto Sans SC 正文使用 `400`，按钮、页签、标签和普通数值使用 `500`，重要标题和关键数值使用 `600`。
- `700` 仅用于危险警告、结算结果或极短的强调，不用于整段文字。
- 不使用伪粗体、描边膨胀或水平拉伸。需要的字重必须随字体资源实际打包。
- 不用全局降低透明度表达次要信息；次级文本仍需满足背景上的可读对比度。

## 行距、字距与排版

- 正文行高为字号的 `1.55` 至 `1.70` 倍；连续叙事和对话优先 `1.65`。
- 按钮、页签和单行标签行高为字号的 `1.25` 至 `1.40` 倍，并通过控件内边距保证点击区域，不靠压缩行高省空间。
- 标题行高为字号的 `1.15` 至 `1.30` 倍；超过一行时改用 Noto Sans SC 或重写标题，避免宋体多行密排。
- 中文默认字距为 `0`。仅 `type.display` 可在专门视觉验收后增加不超过 `0.04em` 的正字距。
- 中英文、数字与单位之间采用统一的排版规则；资源数值和百分比不得因字形宽度变化造成控件跳动。
- 数值控件预留稳定宽度并右对齐；同组数值的小数位、单位位置和正负号格式保持一致。
- 禁止用发光、模糊、重阴影或高强度纹理填充正文。展示标题的特效也必须保留清晰实体字形。

## 720p 与无障碍验收

- 1280 x 720 下逐页检查标题、正文、按钮、数值、悬浮提示、战斗反馈和存档界面，不允许裁字、重叠、被特效遮挡或依赖悬浮才能读全。
- 正文和操作文本对背景的对比度至少达到 `4.5:1`；22 px 及以上的粗体展示文字至少达到 `3:1`。
- 文本容器必须支持中文自然换行，并为最长的事件名、能力来源和本地 AI 文本预留边界。
- 最多两行的按钮应保持固定高度体系；超过容量时改写文案或扩大布局，不缩小字号。
- 高亮、禁用、受伤和压力等状态不能只依靠字形颜色区分，还需配合图标、形状或明确标签。
- 截图验收同时覆盖 1280 x 720、1440 x 900 和高 DPI 缩放；字体边缘需清晰，不能出现缺字方框、错误回退或笔画粘连。

## 授权与打包

- Noto Serif SC 与 Noto Sans SC 使用发布版本中附带的 SIL Open Font License 1.1；引入前记录准确版本、来源 URL 和文件校验值。
- 游戏包中保留字体对应的许可证文本和版权声明，发布说明列出所用字体及授权，不删除或改写字体保留名称。
- 只打包实际使用的 `400`、`500`、`600`、`700` 字重。若选择可变字体，需验证 Godot 导入、Windows 导出和各目标显卡上的字重一致性。
- 若进行字体子集化，必须保留简体中文常用字、项目全部静态文案、角色与地名、标点、全角字符、拉丁字母、阿拉伯数字和常用符号，并为本地 AI 输出配置完整字体回退。
- 子集化流程必须可重复生成并纳入构建验证；不能依赖开发机已安装字体，也不能在运行时从网络下载字体。
- Godot 的 Theme 统一引用仓库内字体资源和回退链。导出包需在未安装 Noto 字体的干净 Windows 环境中验证。

## Logo 与品牌字

- 《问道长生》Logo 是独立设计和导出的品牌资产，不直接依赖 Noto Serif SC、Noto Sans SC 或其他 UI 字体拼出。
- Logo 需要保留可编辑母版、矢量或高分辨率源文件、深浅背景版本和小尺寸简化版本，并单独确认字形创作与商业授权。
- Logo 的艺术字不进入正文、按钮或标题 token；界面中出现产品名时，仍按对应 UI 层级使用规定字体。

## 实施门槛

1. 建立 Godot 全局 Theme 字体资源、回退链、字号 token 和字重映射，禁止页面脚本散落硬编码字号。
2. 完成静态文案字库扫描、本地 AI 回退验证、字体授权清单和离线打包验证。
3. 对主界面、人物、世界、战斗、秘境、轮回、存档和结算页进行 720p 实机截图验收。
4. 只有在缺字率为零、关键文本无裁切、正文对比度达标且长时间阅读无明显发虚后，才能替换现有产品字体。
